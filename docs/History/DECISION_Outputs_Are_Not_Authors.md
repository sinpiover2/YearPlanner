# Outputs Are Not Authors

Date: 2026-07-03

## Decision

Planning should occur exactly once.

Downstream systems should present instructional information without becoming duplicate planning tools.

## Principle

Information is authored once and reused many times.

Every instructional artifact has a single source of truth.

## Source of Truth

Forecast

- Pacing
- Strategic awareness

Units

- Curriculum structure
- Unit purpose

Lesson Planner

- Daily instructional plan
- Agenda
- Materials
- Homework
- Deliverables
- Teacher notes
- Reflection

## Output Channels

Output channels consume information but do not author it.

Examples include:

- Monday Manager
- Daily printouts
- Parent summaries
- Substitute plans
- LMS exports
- Future AI summaries

## Monday Manager

Monday Manager is not a planning application.

It is a beautifully formatted publication.

Its purpose is to answer one question for students:

> What do I need to do this week?

It should derive its information from Lesson Planner rather than maintaining a separate planning workflow.

## Implications

- No duplicate data entry.
- Every piece of instructional information has one owner.
- Outputs remain simple.
- Future integrations become significantly easier because they consume a common instructional record.
