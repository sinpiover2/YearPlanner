# Sprint 4.5 Notes — Real Lesson Authoring

## Sprint objective

Use Lesson Planner as a teacher rather than design it as software.

Improvements should be earned through authentic lesson authoring.

## Development loop

Observe.

Capture.

Categorize.

Automate documentation changes.

Review the diff.

Commit.

## End-of-session practice

At the end of a meaningful authoring session:

1. Record what the teacher naturally attempted.
2. Record where the software interrupted thought.
3. Separate repeated workflow patterns from one-time preferences.
4. Classify discoveries as:
   - architecture
   - interaction philosophy
   - implementation decision
   - unresolved research observation
5. Update enduring documents only after the behavior has been observed clearly enough to justify it.

## Confirmed Sprint 4.5 discoveries

- Lesson Planner must preserve authored work automatically.
- Teachers should not need to think about saving.
- Instructional actions remain the primary organizing object.
- Teachers first build the lesson and then enrich it.
- Writing should precede categorization.
- Collapsing an action marks a shift of attention.
- Revision is normal and should feel safe.
- Lesson Sessions own enactment.
- Curriculum Lessons own intent.
- Changes to a Lesson Session should not silently alter curriculum.
- Updating curriculum should require an explicit, intentional action.
- Deliverables are first-class instructional objects.
- Deliverable due dates are optional.
- Lesson Planner is an instructional outliner and trusted notebook.

## Current development focus

Make Lesson Planner feel like an outliner that can be used comfortably for an extended planning session.

Near-term emphasis:

- uninterrupted writing
- effortless revision
- rapid creation of instructional actions
- easy movement between title and notes
- reordering
- quiet collapse and expansion
- minimal interface chrome

## Deferred until earned through use

- curriculum import from Units
- nested actions
- rich-text formatting beyond minimal emphasis
- Deliverable authoring
- due-date interactions
- Teacher Moves editing
- Evidence editing
- curriculum publishing workflow
